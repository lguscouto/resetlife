# ResetLife 0.7.0

**Build gerado em 14 de julho de 2026**

## Objetivo

1. Ocultar as abas Bem-estar e Revisão durante o onboarding.
2. Indicar quando o check-in do dia já foi feito.

## Implementado

### Navegação consciente do onboarding
- `ResetLifeNavigationBar` recebe `onboardingCompleted` e filtra os destinos.
- Durante o onboarding: apenas `Hoje` e `Onboarding` aparecem na barra inferior.
- Após concluir: todas as 5 abas (Hoje, Organizar, Revisão, Onboarding, Bem-estar).

### Indicador de check-in diário
- `WellbeingCheckInDao.observeDate(date)` adicionado (Flow).
- `WellbeingRepository.observeDate(date)` expõe o check-in do dia.
- `CheckInViewModel` observa o check-in de hoje e define `alreadyDoneToday`.
- `CheckInScreen` mostra mensagem de sucesso "Você já fez o check-in de hoje" e desabilita o botão "Salvar check-in" (ainda é possível editar e salvar de novo manualmente no futuro).
- String `checkin_already_today` em PT-BR.

## Validação

- Unit tests: 28/28 passaram.
- APK instalado no emulador Pixel_8 (versionCode=19).
- Dump de UI confirma: durante onboarding só `['Hoje', 'Onboarding']`; após concluir, todas as 5 abas.
- Check-in salvo → mensagem "já fez o check-in de hoje" aparece e botão salvar desabilita.

## Limitações conhecidas

- Revisão semanal não bloqueia segunda entrada no mesmo período (ainda não há `periodStart` validado no save).
- Check-in permite re-salvar (não há trava rígida de edição), mas o botão inicial fica desabilitado.
