# ResetLife 0.9.1

**Build gerado em 14 de julho de 2026**

## Objetivo

Trocar os ícones Unicode da barra inferior e dos cards dos hubs por ícones vetoriais do Material (crisp em qualquer densidade) e criar um ícone de aplicativo (adaptive icon) próprio do ResetLife.

## Alterado

### Barra inferior (ResetLifeNavigationBar)
- `destination.symbol: String` (Unicode) substituído por `destination.icon: ImageVector`.
- Ícones Material: Hoje = `Today`, Organizar = `Reorder`, Vida = `Favorite`, Perfil = `Person`.
- `Icon(imageVector = destination.icon, contentDescription = description)` em vez de `Text(symbol)`.

### Hubs (LifeScreen / ProfileScreen)
- `AccessCard` recebe `icon: ImageVector` e renderiza `Icon(...)` + título/descrição em `Row`.
- Ícones: Hábitos = `CheckCircle`, Ambiente = `Eco`, Bem-estar = `SelfImprovement`, Revisão = `AutoAwesome`, Configurações = `Settings`.

### Ícone do app
- `res/drawable/ic_launcher_foreground.xml` (vetorial: seta de renovação em branco).
- `res/drawable/ic_launcher_background.xml` (teal `#00695C`).
- `res/mipmap-anydpi-v26/ic_launcher.xml` + `ic_launcher_round.xml` (adaptive icon).
- `AndroidManifest.xml`: `android:icon` e `android:roundIcon` apontam para `@mipmap/ic_launcher`.

### Dependências
- Adicionado `androidx.compose.material:material-icons-extended` (ícones vetoriais).

### Versionamento
- `versionCode = 23`, `versionName = 0.9.1`.

## Validação

- Build `assembleDebug` OK (23/0.9.1).
- Unit tests JVM: 29/29 passaram.
- Dump de tela confirma que os símbolos Unicode (`☀ ▤ ♥ ◔ ✓ ⌂ ❤ ☷`) **não aparecem mais**; barra mantém os 4 rótulos (Hoje/Organizar/Vida/Perfil).
- APK contém `ic_launcher_foreground/background.xml` e `mipmap-anydpi-v26/ic_launcher(_round).xml`; manifest referencia o ícone.
- Onboarding concluído em emulador e tela Hoje abre com a barra (ícones vetoriais).

## Notas

- Sem mudança de navegação ou escopo — apenas camada visual.
- `content-desc` de acessibilidade mantido por destino (`today_nav_description`, etc.).
