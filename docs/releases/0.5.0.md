# ResetLife 0.5.0

**Build gerado em 14 de julho de 2026**

## Objetivo

Entregar o check-in diário de bem-estar e corrigir o fluxo de onboarding para que ele conduza o usuário até a tela Hoje.

## Implementado

### Onboarding (correções)
- Etapa 1 (área) e etapa 2 (disponibilidade) agora têm botão "Próximo".
- `OnboardingViewModel.init` não sobrescreve mais a área selecionada durante um onboarding novo (bug que impedia concluir).
- `AppScreen` navega automaticamente para Hoje após `onboardingCompleted = true` (LaunchedEffect).
- Fluxo validado end-to-end no emulador Pixel_8: área → disponibilidade → duração → Hoje.

### Check-in de bem-estar (0.5.0)
- `WellbeingCheckInEntity` + DAO (tabela `wellbeing_checkin`, Room v4).
- `WellbeingCheckIn` (domain) com mapeadores.
- `WellbeingRepository` (observeAll / get / upsert).
- `CheckInViewModel` + `CheckInViewModelFactory`.
- `CheckInScreen` com sliders 1–5 para humor, energia, estresse e sono.
- `ResetLifeDatabase` atualizado para version = 4 com AutoMigration 3→4.
- `ResetLifeApplication` injeta `wellbeingStore`.
- Destino `Wellbeing` já existe na barra inferior; `CheckInScreen` integrado no `AppScreen`.

## Validação

- Unit tests: 28/28 passaram.
- APK instalado no emulador Pixel_8 (versionCode=16).
- Fluxo de onboarding completo executado via UiAutomator (cliques reais): conclui e abre Hoje.
- Tela Bem-estar abre com sliders e botão "Salvar check-in"; sem crash no logcat.

## Limitações conhecidas

- A aba Bem-estar aparece durante o onboarding (deveria ficar oculta até concluir).
- Check-in não bloqueia segunda entrada no mesmo dia (ainda não há lógica de "já feito hoje").
- Revisão semanal (próximo módulo do escopo) ainda não implementada.
