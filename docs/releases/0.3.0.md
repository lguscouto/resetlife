# ResetLife `0.3.0` — Tarefas e projetos simples

**Data da validação:** 12/07/2026  
**Status:** build debug gerado, testes unitários executados e migração Room validada no emulador  
**Version code:** `3`  
**Version name:** `0.3.0`

## Resumo

A versão `0.3.0` adiciona a área **Organizar**, com projetos e tarefas simples persistidos localmente. O usuário pode criar projetos, associar tarefas, informar nota, prazo e duração, buscar por título, concluir/reabrir e promover uma tarefa para as prioridades de Hoje.

## O que foi entregue

- modelos de domínio `Project` e `Task`;
- status de tarefa: próxima ação, agendada, algum dia e concluída;
- criação com validação de título;
- conclusão e reabertura sem perder metadados;
- busca local por título, ignorando maiúsculas/minúsculas e espaços externos;
- entidade Room `ProjectEntity`;
- entidade Room `TaskEntity`;
- `ProjectDao` e `TaskDao`;
- banco Room atualizado para a versão 2;
- auto-migration Room v1 → v2;
- `OrganizeRepository` e contrato `OrganizeStore`;
- `OrganizeViewModel` com estados de carregamento, erro e feedback;
- tela **Organizar**;
- navegação inferior entre **Hoje** e **Organizar**;
- seleção de projeto ao criar tarefa;
- promoção de tarefa para Hoje respeitando o limite de três prioridades;
- teste unitário da migração usando `MigrationTestHelper`;
- schemas Room 1 e 2 versionados.

## Limitações desta release

- não há colaboração, kanban, anexos, recorrência ou calendário externo;
- não há sincronização em nuvem ou backend;
- o teste instrumentado cobre a migração Room, mas ainda não cobre interação completa da tela Organizar;
- a validação manual confirmou instalação e inicialização; a interação completa de criar projeto/tarefa ainda deve ser adicionada a testes de UI;
- o APK gerado é debug instalável, não um artefato release assinado para publicação na Google Play.

## Avisos não bloqueantes

A compilação ainda informa avisos conhecidos do ambiente:

- KAPT faz fallback da linguagem Kotlin 2.0 para 1.9;
- Gradle informa funcionalidades depreciadas para versões futuras;
- argumentos internos do Room aparecem como não reconhecidos nos source sets de teste, sem impedir a geração dos schemas ou a execução da migração.

Nenhum desses avisos impediu os testes unitários, a geração dos APKs ou a validação da migração no emulador.
