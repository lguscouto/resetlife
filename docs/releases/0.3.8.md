# ResetLife 0.3.8

**Build gerado em 13 de julho de 2026**

## Correção crítica de persistência

- **Bug corrigido**: tarefas criadas via UI não eram persistidas corretamente no banco de dados. O `OrganizeViewModel.addTask()` usava `viewModelScope.launch` sem garantir que a escrita no Room fosse concluída antes que o feedback fosse emitido. Isso gerava o texto "Tarefa criada." mas a tarefa não aparecia na lista.
- **Solução**: mudança explícita para `Dispatchers.IO` dentro do `launch`, garantindo que o `taskDao.upsert()` execute em thread de I/O e complete antes do feedback.
