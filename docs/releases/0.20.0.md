# ResetLife 0.20.0 — Roadmap de Melhorias de UI (0.11 → 0.20)

Este release entrega o plano de melhorias de UI/UX derivado da avaliação de concorrentes
(Streaks, Habi, Habitica, Finch, Done, Super Productivity). Foram 10 versões incrementais,
todas com build verde, testes unitários verdes e push para `origin/master`.

| Versão | Melhoria | Prioridade |
|---|---|---|
| 0.11 | Widgets da home (Hoje + Hábitos) | 🔴 |
| 0.12 | Heatmap de streak (calendário mensal no detalhe) | 🔴 |
| 0.13 | Modo escuro / troca de tema (Claro/Escuro/Sistema) | 🔴 |
| 0.14 | Mensagens de recompensa ao concluir | 🔴 |
| 0.15 | Cores por hábito (seletor de 8 cores) | 🟡 |
| 0.16 | Hábitos negativos (tipo HABIT / AVOID) | 🟡 |
| 0.17 | Backup / exportação de dados em JSON | 🟡 |
| 0.18 | Lembretes locais (notificações via AlarmManager) | 🟡 |
| 0.19 | Estados vazios acionáveis + acessibilidade | 🟢 |
| 0.20 | Modo "sem pressão" de streak (estilo Finch) | 🟢 |

## Detalhes por versão

### 0.11 — Widgets (Jetpack Glance)
- `TodayGlanceWidget`: lista até 3 prioridades do dia com botão "Concluir" (offline, Room).
- `HabitsGlanceWidget`: lista até 4 hábitos com botão de concluir/feito.
- Concluir no widget reflete no app (compartilha `ResetLifeApplication.priorityStore`/`habitStore`).
- `ActionCallback` isolado em `WidgetActions` para testabilidade.
- Teste unitário `WidgetActionTest`.

### 0.12 — Heatmap de streak
- `HabitDetailScreen` acessível clicando no hábito na lista.
- `StreakCalendar` (grid 7×5 = 35 dias) com células concluídas em cor de destaque.
- `HabitDetailViewModel` calcula `streakCurrent` (dias consecutivos) e `completionRate`.
- Teste `HabitDetailViewModelTest` (fake de `HabitRepository`).

### 0.13 — Modo escuro
- `ThemeMode` enum (LIGHT/DARK/SYSTEM); `ResetLifeTheme(themeMode)` aplica `isSystemInDarkTheme()`.
- `ThemeManager` + `DataStoreThemePreferenceStorage` (persistido).
- Bottom sheet "Aparência" no Perfil com 3 opções (RadioButton).
- Teste `ThemeManagerTest` (storage in-memory).

### 0.14 — Mensagens de recompensa
- `RewardProvider` com frases rotativas PT-BR por tipo (prioridade/hábito/tudo feito).
- `RewardMessage` celebrativo (ícone + cor de sucesso).
- Hoje: recompensa ao adicionar/concluir e ao zerar o dia.
- Hábitos: recompensa ao concluir (auto-limpa em 3s).
- Teste `RewardProviderTest`.

### 0.15 — Cores por hábito
- `colorHex` opcional no hábito (entidade/domain/mappers/repo/vm).
- Seletor de 8 cores PT-BR no diálogo de novo hábito.
- Cor aplicada na `Checkbox`/indicador da lista e no heatmap de streak.
- Room AutoMigration 8 → 9.
- Testes `HabitColorParseTest` + `HabitMappersColorTest`.

### 0.16 — Hábitos negativos
- `HabitType` enum (HABIT/AVOID); `type` na entidade (default HABIT).
- Toggle "Hábito"/"Evitar" no diálogo; rótulo "Evitei" e cor de aviso para AVOID.
- Badge de tipo no detalhe.
- Room AutoMigration 9 → 10.
- Teste `HabitTypeTest`.

### 0.17 — Backup / exportação JSON
- `DataExportService` monta JSON (org.json) de todos os repositórios.
- Botão "Exportar" no Perfil via `CreateDocument` (salva onde o usuário quiser).
- `DataExportViewModel` + factory; teste `DataExportServiceTest` (fake source).
- Import não incluso nesta versão (evita conflito de chaves no Room).

### 0.18 — Lembretes locais
- `ReminderPreferences` (DataStore) + `ReminderScheduler` (AlarmManager exact).
- `ReminderReceiver` dispara notificação local (channel próprio, sem som/vibração).
- Bottom sheet "Lembretes" no Perfil (Switch + horário HH:MM + permissão POST_NOTIFICATIONS).
- Teste `ReminderSchedulerTest` (parser de hora).

### 0.19 — Estados vazios + acessibilidade
- `ResetLifeEmptyState` reutilizável (ícone + título + descrição + ação).
- Microcopy acionável em telas que listam dados (organizar/ambiente/listas/bem-estar/revisão).
- `contentDescription` nos ícones de navegação (já existiam; confirmado via teste).
- Teste de strings de empty state.

### 0.20 — Modo sem pressão
- `PressurePreferences` (DataStore) + toggle no detalhe do hábito.
- Quando ativo: oculta a contagem de "Sequência" e mostra mensagem gentil ("Cada dia conta. Vá no seu ritmo."). O heatmap permanece visível.
- Teste `PressurePreferencesTest` (fake in-memory).
