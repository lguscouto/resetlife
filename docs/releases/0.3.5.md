# ResetLife `0.3.5` — Formulários, teclado e prevenção de erros

**Data da validação:** 13/07/2026  
**Status:** build debug gerado, testes unitários, pipeline instrumentado e jornada manual validados  
**Version code:** `8`  
**Version name:** `0.3.5`

## Resumo

A versão `0.3.5` reduz o atrito de entrada na jornada Organizar e padroniza todos os prazos apresentados ao usuário no formato brasileiro **`DD/MM/AAAA`**. A interface usa máscara visual para entrada numérica e mantém a persistência interna em ISO (`AAAA-MM-DD`) para preservar a compatibilidade com o banco existente.

## O que foi entregue

- validação inline de título obrigatório;
- validação inline de data impossível ou incompleta;
- validação inline de duração vazia inválida, não numérica ou não positiva;
- mensagens de erro próximas ao campo que originou o problema;
- preservação do conteúdo digitado ao rejeitar um formulário;
- limpeza do erro ao editar o campo correspondente;
- entrada de prazo com teclado numérico e transformação visual `DD/MM/AAAA`;
- aceitação da data brasileira completa na camada de estado (`15/07/2026`);
- conversão interna para `2026-07-15` somente no momento da criação da tarefa;
- conversão de valores persistidos para `DD/MM/AAAA` na lista de tarefas;
- rejeição explícita de valores ISO como entrada de interface;
- foco sequencial com `ImeAction.Next`;
- `ImeAction.Done` para concluir projeto, tarefa e prioridade;
- `ImeAction.Search` para a busca;
- `imePadding` e rolagem para manter ações acessíveis com o teclado aberto;
- flags de salvamento para impedir submissão duplicada;
- limpeza de rascunho somente após persistência bem-sucedida;
- feedback visual de sucesso para projeto, tarefa e prioridade criados;
- testes unitários de formatação, normalização, datas inválidas, duração inválida e duplo envio;
- nenhuma alteração em entidades, migrações Room ou contratos persistentes.

## Formato de data

| Camada | Formato |
|---|---|
| Entrada e apresentação na interface | `DD/MM/AAAA` |
| Exemplo visível | `15/07/2026` |
| Persistência interna compatível | `AAAA-MM-DD` |

A máscara visual permite digitar apenas números no teclado numérico, sem reordenar os dígitos durante a inserção. As barras são apresentadas automaticamente na tela.

### Validação manual da jornada Organizar

No emulador, foram executados os passos:

1. abrir Organizar;
2. digitar uma tarefa de teste;
3. digitar `30/02/2026` usando o teclado numérico;
4. digitar duração `0`;
5. tocar em `Criar tarefa`;
6. confirmar os dois erros inline:
   - `Data inválida. Use DD/MM/AAAA.`;
   - `A duração deve ser um número inteiro positivo.`;
7. corrigir para `15/07/2026` e `30` sem reiniciar o formulário;
8. confirmar `Tarefa criada.`;
9. confirmar a tarefa em `Abertas`;
10. confirmar o metadado `15/07/2026 • 30 min`;
11. confirmar que nenhum valor ISO (`2026-07-15`) é apresentado na interface;
12. criar um rascunho e alternar Hoje → Organizar;
13. confirmar a preservação do rascunho.
